# DB-ATCNet — FPGA Inference Specification

**Audience:** the RTL team implementing the accelerator in [`../fpga/`](../fpga/README.md).
**Scope:** the frozen inference graph, its layer-by-layer tensor-shape flow, the fixed-point
convention, and the operator ↔ RTL-module mapping. This is the contract the SystemVerilog
implementation is verified against (bit-exact vs. the Python golden model).

> Source of truth for hyperparameters: [`configs/config.yaml`](configs/config.yaml).
> Source of truth for the graph: `DB_ATCNet(...)` and the ADBC conv block in
> [`DB-ATCNet/models.py`](DB-ATCNet/models.py). Any change there must be reflected here **and**
> re-golden'd for the RTL testbenches in [`../fpga/sim/`](../fpga/sim/README.md).

---

## 1. Deployment target

| Item | Value |
|---|---|
| Device | Xilinx Zynq UltraScale+ **`xczu7ev-ffvc1156-2`** (ZCU104 / ZCU106) |
| Task | Binary motor imagery — **class 0 = Right Hand, class 1 = Left Leg** (HaLT markers 2 / 4) |
| Input window | 5 selected EEG channels × 600 samples @ 200 Hz (3.0 s) |
| Output | 1-bit class (`0x00` / `0x01`) via the AXI-Lite status/class mailbox |
| Arithmetic | **Q8.8** fixed-point (16-bit signed), bit-exact vs. Python golden |
| Weights | Subject-dependent, trained offline, loaded over AXI-Lite at boot |
| Latency budget | ≤ 4 ms (one EEG window); measured core inference is **sub-millisecond** |

---

## 2. Fixed-point convention

All datapath signals are **Q8.8**: 16-bit signed, 8 integer / 8 fractional bits (range
≈ [−128, +127.996], resolution 2⁻⁸ ≈ 0.0039). Rules:

- Multiplies accumulate in wider precision; **saturate + round back to Q8.8 at every stage
  boundary** (`../fpga/rtl/conv/sat_add.sv`).
- Non-linearities (**ELU**, **sigmoid**) are precomputed Q8.8 lookup tables in
  `../fpga/data/lut/`, generated by `../fpga/scripts/gen_*_lut.py`.
- Every intermediate tensor has a matching `.hex` golden file under `../fpga/data/golden_q88/`
  that the per-module testbenches assert against.

---

## 3. I/O contract

**Input** — one inference consumes a single window streamed over AXI4-Stream:

```
raw EEG   : 19 channels × 600 samples, Q8.8, little-endian  (6000 bytes = 3000 × 2)
selected  : 5 channels chosen by the channel-selection front-end (§6)
```

**Output** — a single class byte written to the mailbox once the pipeline drains.

## 4. Layer-by-layer tensor-shape flow

Shapes below are the **golden-model reference** (logical `features × channels × time`); the RTL is
streaming and bit-exact to these. Hyperparameters: `F1=16`, branch depth multipliers `D∈{2,4}`,
`F2 = F1·D`, temporal kernel `kernLength=64`, separable kernel `16`, `poolSize=7`, `n_windows=5`,
window length `6`, `n_classes=2`.

| # | Stage | Operator | Out shape (feat × ch × time) | RTL module |
|---|---|---|---|---|
| 0 | Channel select | 5-of-19 mask (static or Gumbel) | 1 × 5 × 600 | `window/gumbel_channel_adapter` |
| 1 | Temporal conv | `Conv2D(F1=16, 64×1, same)`, no bias | 16 × 5 × 600 | `conv/conv2d_temporal` |
| 2 | ECA₁ | Efficient Channel Attention gate | 16 × 5 × 600 | `attention/eca1_pipeline` (`eca_attention`, `gap_accumulator`, `gate_apply`) |
| 3a | Branch A | `DepthwiseConv2D(1×5, D=2)` → BN → ELU | 32 × 1 × 600 | `conv/depthwise_spatial` + `conv/elu` |
| 3b | Branch A | `AvgPool(8×1)` → `Conv2D(F2,16×1)` → BN → ELU → `AvgPool(7×1)` | 32 × 1 × ~10 | `conv/avg_pool_time`, `conv/branch_pipeline` |
| 4a | Branch B | `DepthwiseConv2D(1×5, D=4)` → BN → ELU | 64 × 1 × 600 | `conv/depthwise_spatial` + `conv/elu` |
| 4b | Branch B | `AvgPool(8×1)` → `Conv2D(F2,16×1)` → BN → ELU → `AvgPool(7×1)` | 64 × 1 × ~10 | `conv/avg_pool_time`, `conv/branch_pipeline` |
| 5 | Fuse | concat(Branch A, Branch B) along features | C_f × ~10 | `db_atcnet_post_eca1` |
| 6 | ECA₂ | Efficient Channel Attention gate | C_f × ~10 | `attention/eca_attention` |
| 7 | CBAM (per window) | channel attention (dual-pool → shared MLP → sigmoid) | C_f × 6 | `attention/cbam_channel_attn` |
| 8 | CBAM (per window) | spatial attention (tri-pool → `Conv(7×1)` → sigmoid) | C_f × 6 | `attention/cbam_spatial_attn` |
| 9 | TCFN | dilated **causal** conv, `depth=2`, `kernel=4`, `filters=32`, dilation {1,2}, ELU, residual | 32 × 6 | `conv/tcfn` |
| 10 | Window head | `Dense(2)` per window (×5 sliding windows) | 2 (×5) | `classifier/dense_classifier` |
| 11 | Fusion | average the 5 window logits → softmax → argmax | 1-bit class | `classifier/output_head` |

**Temporal reduction:** 600 → `AvgPool(8)` → 75 → `AvgPool(7)` → **≈10 steps** (~50 ms/step).
**Sliding-window fusion:** five overlapping length-6 windows are extracted from the 10-step
sequence (starts 0…4), each classified independently; the five 2-logit vectors are averaged before
the final softmax.

RTL integrator grouping (front→back): `db_atcnet_conv2d_eca1` (rows 1–2) → `db_atcnet_post_eca1`
(rows 3–6) → `db_atcnet_window_pipeline` (rows 7–11). Top: `db_atcnet_top` (raw EEG → class) wrapped
by `db_atcnet_axi` for deployment.

## 5. Operator notes for RTL

- **Conv2D temporal (row 1)** and **CBAM channel dense (row 7)** are the DSP-dominant stages
  (~732 and ~608 DSP48E2 respectively) — they must retain their DSPs; see
  [`../fpga/synth/README.md`](../fpga/synth/README.md).
- **CBAM replaces MHA** deliberately: only fixed-size global pooling + a small shared MLP + a 7×1
  conv, all O(C·H·W) and statically schedulable (no variable-length softmax). See the Stage-1
  rationale in [`README.md`](README.md).
- **TCFN is causal** — left-padding only, no lookahead buffer.
- **BatchNorm** folds into the preceding conv's scale/bias at export (no runtime BN on-chip).
- **ELU / sigmoid** are Q8.8 LUTs, not computed.

## 6. Channel-selection modes

Both modes present an identical downstream datapath; only the front-end mask differs.

| Mode | Channels | Loading | Notes |
|---|---|---|---|
| Static (ablation-ranked) | `[Cz, P3, T5, F7, T3]` (idx `[17, 6, 14, 10, 12]`) | hardwired 5-of-19 MUX | no runtime config |
| Static (weight-magnitude) | `[T5, T3, Fp2, Fz, Pz]` | hardwired 5-of-19 MUX | no runtime config |
| **Gumbel (per-subject)** | learned; deployed set e.g. `[Cz, P3, T5, F7, F8]` | 25-bit register from flash LUT at boot | `window/gumbel_channel_adapter` |

Per-subject Gumbel indices are exported as JSON and packed into the boot LUT.

---

## 7. Weight layout & loading

- Exported per subject/configuration from the trained HDF5 model to per-layer numpy arrays, then
  packed to Q8.8 `.hex` banks (`../fpga/weights/`, git-ignored/regenerable).
- Loaded over **AXI4-Lite** into self-decoding `weights/weight_bank` blocks via
  `weights/axi_lite_weight_loader` at boot; the inference datapath then streams from BRAM.

## 8. Resource & timing budget (final impl, `xczu7ev`, OOC core)

| Metric | Value |
|---|---|
| LUTs | ~193k / 230k (≈84 %) |
| DSP48E2 | ~1,727 / 1,728 (≈99.5 %) — **binding constraint** |
| Block RAM | ~80 / 312 tiles |
| Timing | closes at 50–60 MHz (positive WNS) |
| On-chip power | ≈5.1 W (full PS+PL) |
| Core inference | sub-millisecond |
