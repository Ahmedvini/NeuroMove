# `sim/` — testbenches & bit-exact verification

Every compute module and every top-level integrator has a self-checking
testbench that compares the RTL output against a **Q8.8 golden `.hex`** file
generated by the Python model in [`../scripts/`](../scripts/README.md). A test
passes only on a **bit-exact** match.

## Running tests

```bash
# Required once per shell so xsim can link against the C runtime:
export LIBRARY_PATH=/usr/lib/x86_64-linux-gnu:$LIBRARY_PATH

./run_vivado_tests.sh <test_name>      # Linux
./run_vivado_tests.ps1 <test_name>     # Windows (PowerShell)
```

The runner compiles the DUT + its `*_tb.sv`, runs `xsim`, and reports
pass/fail. Golden hex paths are passed through `-generic *_FILE=...` so
`$readmemh` resolves; if a test shows large mismatches, first confirm the log
says *"data file … read successfully"*.

## Test-name cheat sheet

| Name(s) | DUT |
|---|---|
| `conv2d_temporal`, `c2e1` | temporal conv (+ conv2d→eca1) |
| `eca1_pipe`, `eca2_pipe` | ECA attention pipelines |
| `branchA_pipe`, `branchB_pipe` | dual depthwise-separable branches |
| `chan_w0`, `chan_all` | CBAM channel attention |
| `spat_w0`, `spat_all` | CBAM spatial attention |
| `tcfn_w0`, `tcfn_all` | TCFN feed-forward |
| `dense_w0`, `head` | dense classifier / output head |
| `dwise_a`, `dwise_b`, `sep_a_tm`, `sep_b_tm` | depthwise / separable temporal |
| `gap`, `gate_apply`, `sat_add`, `elu`, `pool1_a`, `pool2_a`, `divider` | primitives |
| `pipe` | `db_atcnet_window_pipeline` (end-to-end back half) |
| `post_eca1`, `top`, `axil`, `gca` | post-ECA₁, full top, AXI-Lite loader, Gumbel adapter |

(See the `case`/dispatch table in `run_vivado_tests.sh` for the authoritative
name → DUT+tb mapping.)

## Notable testbenches

- `db_atcnet_top_tb.sv` — raw EEG → class, full network.
- `db_atcnet_axi_tb.sv` — exercises the AXI-Stream + AXI-Lite deployment wrapper.
- `db_atcnet_window_pipeline_tb.sv` — attention + classifier back half.
- `aes_256_gcm_tb.sv`, `hmac_sha256_tb.sv` — security cores.
- `eeg_dataset_tb.sv` — dataset/config path.
