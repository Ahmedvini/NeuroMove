# DB-ATCNet — FPGA Accelerator for EEG Motor-Imagery Classification

A SystemVerilog hardware accelerator that decodes motor-imagery EEG into a
movement class, designed and verified for the **Xilinx Zynq UltraScale+
`xczu7ev`** (ZCU104 / ZCU106). It is the programmable-logic core of a
brain–computer interface (BCI) that drives assistive prostheses: raw EEG goes
in, a 1-bit class comes out, end-to-end on-chip in well under a millisecond.

The model is **DB-ATCNet** — a dual-branch, attention-augmented temporal
convolutional network — implemented as a fully streaming, fixed-point pipeline
that is **bit-exact** against a Python golden model.

## 1. What this repo is

| | |
|---|---|
| **Function** | 2-class motor-imagery decode (e.g. left vs. right) from a 5-of-19-channel EEG window |
| **Target part** | `xczu7ev-ffvc1156-2` (ZCU104 / ZCU106) |
| **Language** | SystemVerilog (synthesizable, Vivado) |
| **Arithmetic** | Q8.8 fixed-point (16-bit signed), bit-exact vs. Python golden |
| **Deployment** | AXI4-Stream data in, AXI4-Lite weights + status/class mailbox |
| **Host stack** | ESP32-S3 firmware + Zynq PS bare-metal bridge + Python bench tools |
| **Weights** | Subject-dependent, trained offline, loaded over AXI-Lite at boot |

The repo is RTL-first: everything needed to **parse, simulate, synthesize,
implement, and deploy** the accelerator lives here. The trained weights and
generated golden tensors are reproducible (and git-ignored) — see
[`scripts/`](scripts/README.md).

## 2. Architecture / dataflow

A single inference consumes one EEG window and streams it through the pipeline
below. Each stage is its own module and has its own bit-exact testbench.

```
 raw EEG (19 ch, Q8.8)
        │
        ▼
 [window]  gumbel_channel_adapter ── selects 5 of 19 channels (learned mask)
        │
        ▼
 [conv]    conv2d_temporal ─────────── temporal conv (2-stage MAC pipeline)
        │
        ▼
 [attn]    eca1_pipeline ───────────── ECA channel attention (gate stream)
        │
        ▼
 [conv]    post_eca1 → branch_pipeline ── dual depthwise-separable branches A/B
        │
        ▼
 [window]  db_atcnet_window_pipeline ── orchestrates the attention block below
        │
        ├─ cbam_channel_attn ────────── CBAM channel attention (Dense1/Dense2)
        ├─ cbam_spatial_attn ────────── CBAM spatial attention
        │
        ▼
 [conv]    tcfn ────────────────────── temporal-convolution feed-forward
        │
        ▼
 [classifier] dense_classifier → output_head ── logits → 1-bit class
        │
        ▼
   class bit (0 / 1)
```

Top-level integrators (smallest to largest scope):

| Module | Scope |
|---|---|
| `rtl/db_atcnet_conv2d_eca1.sv` | raw EEG → ECA₁-gated stream |
| `rtl/db_atcnet_post_eca1.sv` | ECA₁-gated stream → ECA₂ buffer |
| `rtl/db_atcnet_window_pipeline.sv` | ECA₂ buffer → class (attention + classifier) |
| `rtl/db_atcnet_top.sv` | raw EEG → 1-bit class (no AXI) |
| `rtl/db_atcnet_axi.sv` | the deployment wrapper (AXI-Stream + AXI-Lite) |

Full module-by-module map: [`rtl/README.md`](rtl/README.md).

## 3. Fixed-point convention

All datapath signals are **Q8.8** — 16-bit signed, 8 integer / 8 fractional
bits. Multiplies accumulate in wider precision and are saturated + rounded back
to Q8.8 at each stage boundary (`rtl/conv/sat_add.sv`). Non-linearities (ELU,
sigmoid) are precomputed Q8.8 lookup tables in [`data/lut/`](data/README.md),
generated by `scripts/gen_*_lut.py`. Because the hardware is bit-exact, every
intermediate tensor has a matching `.hex` golden file under `data/golden_q88/`
that the testbenches assert against.

## 5. Quick start

### Prerequisites
- **Vivado 2025.2** (xvlog / xelab / synthesis). On Linux export the C runtime
  path so `xsim` can link:
  ```bash
  export LIBRARY_PATH=/usr/lib/x86_64-linux-gnu:$LIBRARY_PATH
  ```
- Plenty of RAM/swap for full synthesis (peaks ~13–20 GB).

### Parse + elaborate the whole design
```bash
./build.sh                 # default top = db_atcnet_axi
./build.sh db_atcnet_top   # alternate top (raw EEG → class, no AXI)
```

### Run the unit / integration tests (bit-exact vs. golden)
```bash
export LIBRARY_PATH=/usr/lib/x86_64-linux-gnu:$LIBRARY_PATH
./sim/run_vivado_tests.sh conv2d_temporal   # one module
./sim/run_vivado_tests.sh pipe              # full window pipeline
./sim/run_vivado_tests.sh top               # raw EEG → class
```
See [`sim/README.md`](sim/README.md) for the full list of test names.

### Synthesize / implement / write a bitstream
```bash
vivado -mode batch -source synth/synth_fit.tcl  -tclargs xczu7ev-ffvc1156-2-e
vivado -mode batch -source synth/impl_fit.tcl
vivado -mode batch -source synth/write_bitstream.tcl
```
See [`synth/README.md`](synth/README.md) for the flow and timing/area notes.

### Deploy (bench)
1. Build the Vivado IPI block design (Zynq PS + AXI-DMA + `db_atcnet_axi`),
   export the `.xsa`, and build the PS app — see [`sw/zynq_ps/README.md`](sw/zynq_ps/README.md).
2. Load bitstream + ELF over JTAG: `scripts/load_zcu106.tcl`.
3. Stream a window from the host or ESP32 — see [`sw/README.md`](sw/README.md).

## 6. End-to-end system

```
 EEG ─► ESP32-S3 ──UART 460800──► Zynq PS (UART1) ──AXI-DMA──► PL: db_atcnet_axi
        (acquire/replay,           (bare-metal bridge,            (inference)
         Q8.8 windowing)            poll status, read class)         │
            ▲                                                        │ 1-bit class
            └────────────── servo command (PCA9685) ◄───────────────┘
```

- The ESP32 sends one **6000-byte** window (3000 × little-endian Q8.8) per
  inference; the PL returns a single class byte (`0x00`/`0x01`).
- Wire protocol, memory map, and register layout: [`sw/zynq_ps/README.md`](sw/zynq_ps/README.md).
- For bench testing without an ESP32, the Python host tools in `sw/host/`
  drive the same UART path.

## 7. Results (final implementation)

Implemented on `xczu7ev` (ZCU104), out-of-context core:

| Metric | Value |
|---|---|
| Fit | ✓ fits ZCU104 / ZCU106 |
| LUTs | ~193k / 230k (≈84 %) |
| DSP48E2 | ~1,727 / 1,728 (≈99.5 %) |
| Block RAM | ~80 / 312 tiles |
| Timing | closes at 50–60 MHz (positive WNS) |
| On-chip power | ≈5.1 W (full PS+PL deployment) |
| Core inference | sub-millisecond (well inside the 4 ms EEG window budget) |

DSP utilization is the binding constraint; the synthesis/area/timing trade-off
history and tuning knobs are documented in [`synth/README.md`](synth/README.md).

## 8. Security stack

`rtl/security/` is an independent set of cryptographic cores built alongside the
AI pipeline for authenticating/encrypting the EEG data and control path:
AES-256-GCM, SHA-256, HMAC-SHA256, RSA-2048, and a secure-boot skeleton. It is
not in the inference datapath; see [`rtl/README.md`](rtl/README.md#security).
